[2026-04-07] Initial Security Review
-------------------------------------
File reviewed: hr-system0407.html (~435 lines, single HTML file)

SECURITY FINDINGS:
  [CRITICAL] Real PII hardcoded in source (names, ID cards, phones, birthdays, emergency contacts)
             - Violates Taiwan Personal Data Protection Act
  [CRITICAL] Plain text passwords stored in JS array (password:"1234")
  [CRITICAL] XSS - innerHTML used with unescaped user input (reason, name, dept fields)
  [CRITICAL] Client-side only authorization - `CU` variable can be modified in browser console
  [CRITICAL] No server-side approval enforcement on workflow

  [MEDIUM]   No data persistence - all state lost on page reload
  [MEDIUM]   No input validation (negative hours, empty fields)
  [MEDIUM]   No CSRF protection, no session management

  [MINOR]    Hardcoded constants (VP_ID, GM_ID, HOURLY_BASE)
  [MINOR]    No HTTPS enforcement
  [MINOR]    No rate limiting on login

ASSESSMENT:
  - Functionally complete for ~20 person company
  - Feature coverage: clock in/out, leave, OT, schedule, approval, comp time - all good
  - Architecture: pure client-side, no backend - NOT safe for production use

RECOMMENDATIONS (phased):
  Phase 1 (must-fix): Remove PII, add backend auth, hash passwords, XSS protection, data persistence
  Phase 2 (architecture): Backend API, rate limiting, audit log, HTTPS
  Phase 3 (features): Leave balance tracking, export/backup, notifications, delegation
